# ⚫ 짭바둑 · 온라인 알까기 ⚪

바닐라 JS 와 Socket.io 로 만든 **실시간 1:1 멀티플레이어 알까기(스톤 플리킹) 게임**입니다.
미니멀한 다크 테마의 단일 페이지에서, 상대와 매칭되어 15초 턴제로 대국합니다.

## 🌐 라이브 데모

**https://zzapbaduk-alkkagi.onrender.com**

> 무료 플랜이라 첫 접속 시 서버가 깨어나는 데 30초~1분 정도 걸릴 수 있습니다.
> 브라우저 탭 2개(또는 친구와 함께)로 접속해 **상대 찾기**를 누르면 대국이 시작됩니다.

## 🎮 게임 규칙

- 각 플레이어는 **돌 3개**(검은돌 vs 흰돌)로 대국합니다. (빠른 승부, 3분 이내)
- **메인 로비**에서 방을 만들거나(방 이름·돌 색 선택 가능) 열린 방에 참가하고, ⚡ 빠른 매칭(색 랜덤)도 가능합니다.
- 대국 전 **돌 배치 단계(30초)**: 자기 진영(판의 절반) 안에 돌 3개를 자유롭게 드래그로 배치합니다.
  - 상대 배치는 대국 시작 전까지 보이지 않아 심리전이 가능합니다.
- 돌을 **드래그했다가 놓으면** 슬링샷처럼 반대 방향으로 튕겨 나갑니다.
  - 드래그하는 동안 **파워/방향 벡터 화살표**가 표시됩니다. (파랑→노랑→빨강 순으로 위험)
  - ⚠️ **너무 세게 치면 내 돌도 판 밖으로 날아갑니다!** (파워 50% 이상으로 빗맞히면 이탈 위험)
- **턴제**이며, 한 턴의 제한 시간은 **15초**입니다. (초과 시 턴이 넘어감) 선공은 항상 검은돌입니다.
- **승리 조건**: 상대의 돌 3개를 모두 판 밖으로 밀어내면 승리합니다.
- 알까기 특성상 게임판에는 벽이 없습니다. 판 밖으로 나간 돌은 탈락합니다.

## 🏗️ 아키텍처

- **프론트엔드**: Vanilla JS, HTML5 Canvas, CSS (단일 페이지, 모던 다크 테마)
- **백엔드**: Node.js, Express, Socket.io (실시간 1:1 방 매칭 및 턴 관리)
- **물리 엔진**: 가벼운 2D 탄성 충돌
  - 돌-돌 충돌: 질량이 같은 완전 탄성 충돌(법선 성분 교환)
  - 마찰로 감속하다가 정지
  - `public/shared/physics.js` 를 **서버와 클라이언트가 공유**하여
    동일한 결정론적 시뮬레이션 결과를 보장합니다. (서버가 권위적으로 최종 계산)

```
zzapbaduk/
├─ server.js               # Express + Socket.io 서버 (매칭/턴/물리 권위 계산)
├─ package.json
├─ public/
│  ├─ index.html           # 단일 페이지 UI
│  ├─ style.css            # 모던 다크 테마
│  ├─ game.js              # 캔버스 렌더링 + 드래그 조작 + 소켓 통신
│  └─ shared/
│     └─ physics.js        # 서버·클라이언트 공용 물리 엔진
└─ README.md
```

## 🚀 로컬 실행 방법

```bash
# 1. 의존성 설치
npm install

# 2. 서버 실행
npm start

# 3. 브라우저에서 접속 (2개의 탭/창으로 각각 접속하면 매칭됩니다)
#    http://localhost:3000
```

> 두 명이 있어야 대국이 시작됩니다. 브라우저 탭 2개를 열고 각각 **상대 찾기**를 누르면 서로 매칭됩니다.

## 📡 통신 이벤트 (Socket.io)

| 이벤트 | 방향 | 설명 |
|--------|------|------|
| `getLobby` / `lobbyUpdate` | C ↔ S | 로비 방 목록 조회/갱신 |
| `createRoom` | C → S | 방 생성 `{name, color: black/white/random}` |
| `roomWaiting` | S → C | 방 생성 완료, 상대 대기 중 |
| `joinRoom` | C → S | 방 입장 `{roomId}` |
| `findMatch` / `waiting` | C ↔ S | 빠른 매칭(색 랜덤) 대기열 |
| `placementStart` | S → C | 돌 배치 단계 시작(내 색/기본 배치/30초 제한) |
| `placeStones` | C → S | 배치 제출 `{positions: [{x,y}×3]}` (서버가 진영·겹침 검증) |
| `opponentPlaced` | S → C | 상대 배치 완료 알림 |
| `gameStart` | S → C | 대국 시작(전체 돌 배치/턴 정보) |
| `shot` | C → S | 돌 발사 `{stoneId, vx, vy}` |
| `shotResult` | S → C | 서버 권위 최종 상태 + 다음 턴 + 승자 |
| `turnTimeout` | S → C | 15초 초과로 턴 넘김 |
| `rematch` | C → S | 재대국(같은 색, 배치부터 다시) |
| `leaveRoom` | C → S | 방 나가기(로비 복귀) |
| `opponentLeft` | S → C | 상대 접속 종료(부전승) |
